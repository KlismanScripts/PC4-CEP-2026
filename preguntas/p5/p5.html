Cuadro Latino 4×4 – Horas de Inactividad (dato faltante)

01
Contexto de Experimento
Cuadrado Latino 4×4 – con un dato faltante (estimación de Yates)

📌 Enunciado: SmartLine Manufacturing busca reducir el tiempo de inactividad no planificada en su planta mediante algoritmos de mantenimiento predictivo. Se evaluaron 4 algoritmos (A: Regresión + reglas expertas, B: Random Forest, C: LSTM, D: Transformer) bajo un diseño de Cuadrado Latino 4×4 que controla (1) la Línea de Producción (antigüedad de maquinaria) y (2) el Turno de operación (carga y disponibilidad técnica). La variable de respuesta es Horas de Inactividad No Planificada al mes (menor es mejor). Durante la recolección, un fallo en la pasarela IoT de la Línea 3 durante el Turno 3 impidió registrar el dato correspondiente al algoritmo A.

A continuación, se presenta la tabla de datos original del experimento con el valor perdido:

Línea \ Turno Turno 1 Turno 2 Turno 3 Turno 4
Línea 124.47 (A)19.69 (B)17.04 (C)22.82 (D)
Línea 221.63 (B)17.32 (C)24.75 (D)25.24 (A)
Línea 316.75 (C)21.54 (D)PERDIDO (A)20.85 (B)
Línea 423.25 (D)23.54 (A)21.66 (B)18.87 (C)

🎯 Objetivo general

Evaluar si existen diferencias estadísticamente significativas en las Horas de Inactividad No Planificada causadas por el algoritmo de mantenimiento (tratamiento), controlando los efectos de bloque de línea de producción y turno. Se estimará el dato faltante mediante el método de Yates y se identificará el algoritmo que minimiza las horas de inactividad.

🧪 Factores estudiados

  • Factor Tratamiento (Algoritmo): 4 niveles (A: Regresión+reglas, B: Random Forest, C: LSTM, D: Transformer).
  • Factor Bloque Fila (Línea de Producción): 4 niveles (Línea 1 a 4).
  • Factor Bloque Columna (Turno): 4 niveles (Turno 1 a 4).
  • Variable de Respuesta: Horas de inactividad no planificada al mes (menor es mejor).
  • Diseño no balanceado: Un dato faltante en la celda (Línea 3, Turno 3, Algoritmo A).
02
Formulación de Hipótesis
Contrastes estadísticos (α = 0.05) y estimación del dato faltante

📐 Estimación del dato perdido (Método de Yates)

Para un cuadrado latino con una sola observación faltante en la posición (fila i, columna j, tratamiento k), la estimación de Yates es:

X̂ = [ r(Ri + Cj + Tk) – 2G ] / [(r–1)(r–2)]

Donde r = 4, Ri es la suma de la fila i (sin el faltante), Cj suma de columna j (sin faltante), Tk suma del tratamiento k (sin faltante), y G suma total de todas las observaciones existentes.

Cálculo: Sumas existentes (sin el dato perdido):

  • Fila 3 (Línea 3): 16.75 + 21.54 + 20.85 = 59.14
  • Columna 3 (Turno 3): 17.04 + 24.75 + 21.66 = 63.45 (faltante de L3,T3 excluido)
  • Tratamiento A: 24.47 (L1,T1) + 25.24 (L2,T4) + 23.54 (L4,T2) = 73.25
  • Suma total G (15 valores): 24.47+19.69+17.04+22.82+21.63+17.32+24.75+25.24+16.75+21.54+20.85+23.25+23.54+21.66+18.87 = 319.42

Aplicando fórmula: X̂ = [4(59.14 + 63.45 + 73.25) – 2·319.42] / [(4–1)(4–2)] = [4(195.84) – 638.84] / (3·2) = (783.36 – 638.84)/6 = 144.52/6 = 24.0867

Por tanto, el valor estimado para la celda perdida (Línea 3, Turno 3, Algoritmo A) es 24.0867 (≈24.09).

⚖️ Hipótesis a Contrastar

🔹 Algoritmo (Tratamiento):
H₀: μA = μB = μC = μD (no hay efecto del algoritmo)
H₁: Al menos un algoritmo genera horas medias de inactividad diferentes.

Interpretación: evaluamos si el tipo de modelo predictivo influye en la inactividad.

🔹 Línea (bloque fila):
H₀: μL1 = μL2 = μL3 = μL4
H₁: Al menos una línea produce inactividad media distinta.
🔹 Turno (bloque columna):
H₀: μTurn1 = μTurn2 = μTurn3 = μTurn4
H₁: Al menos un turno presenta inactividad media diferente.
⚠️ Importante (dato faltante): La presencia de un dato perdido modifica los grados de libertad. En un cuadrado latino completo, GLError = (r-1)(r-2) = 6. Con una estimación, se pierde un grado de libertad adicional, por lo que GLError = (r-1)(r-2)-1 = 5. Esto causa diferencias entre Minitab (que no resta el grado) y los demás softwares (que sí lo hacen). Lo veremos en las tablas ANOVA.

# 1. CONSTRUCCIÓN DE LA MATRIZ DE DATOS
Linea <- factor(rep(paste0("L", 1:4), each = 4))
Turno <- factor(rep(paste0("T", 1:4), times = 4))
Algoritmo <- factor(c("A", "B", "C", "D",
                      "B", "C", "D", "A",
                      "C", "D", "A", "B",
                      "D", "A", "B", "C"))

Horas <- c(24.47, 19.69, 17.04, 22.82,
           21.63, 17.32, 24.75, 25.24,
           16.75, 21.54, NA,    20.85, # L3, T3, Alg A es el dato perdido
           23.25, 23.54, 21.66, 18.87)

df <- data.frame(Linea, Turno, Algoritmo, Horas)

# 2. ESTIMACIÓN ANALÍTICA DEL DATO PERDIDO (Mínimos Cuadrados DCL)
t <- 4
idx_na <- which(is.na(df$Horas))
na_linea <- df$Linea[idx_na]
na_turno <- df$Turno[idx_na]
na_alg <- df$Algoritmo[idx_na]

# Sumas parciales sin el dato faltante
R_sum <- sum(df$Horas[df$Linea == na_linea], na.rm = TRUE)
C_sum <- sum(df$Horas[df$Turno == na_turno], na.rm = TRUE)
T_sum <- sum(df$Horas[df$Algoritmo == na_alg], na.rm = TRUE)
G_sum <- sum(df$Horas, na.rm = TRUE)

# Aplicar la fórmula DCL: Y^ = [t(R+C+T) - 2G] / [(t-1)(t-2)]
y_hat <- (t * (R_sum + C_sum + T_sum) - 2 * G_sum) / ((t - 1) * (t - 2))
df$Horas[idx_na] <- y_hat

cat("--- DATO PERDIDO ESTIMADO ---\n")
cat(sprintf("Línea 3, Turno 3, Algoritmo A = %.4f\n\n", y_hat))

# 3. ANÁLISIS DE VARIANZA (ANOVA)
# Generar modelo inicial
modelo <- aov(Horas ~ Algoritmo + Linea + Turno, data = df)
anova_tab <- anova(modelo)

# Ajustar grados de libertad del Error por el parámetro estimado (-1)
anova_tab["Residuals", "Df"] <- anova_tab["Residuals", "Df"] - 1

# Recalcular Cuadrados Medios (CM), Estadístico F (F0) y Valor-P
anova_tab["Residuals", "Mean Sq"] <- anova_tab["Residuals", "Sum Sq"] / anova_tab["Residuals", "Df"]
mse_adj <- anova_tab["Residuals", "Mean Sq"]
df_res <- anova_tab["Residuals", "Df"]

for(factor in c("Algoritmo", "Linea", "Turno")) {
  anova_tab[factor, "F value"] <- anova_tab[factor, "Mean Sq"] / mse_adj
  anova_tab[factor, "Pr(>F)"] <- pf(anova_tab[factor, "F value"], 
                                    anova_tab[factor, "Df"], 
                                    df_res, 
                                    lower.tail = FALSE)
}

cat("--- TABLA ANOVA (AJUSTADA POR DATO PERDIDO) ---\n")
print(anova_tab)
cat("\n")

# 4. PRUEBAS POST-HOC DE TUKEY EXHAUSTIVAS
# Calcular límite crítico HSD
alfa <- 0.05
q_crit <- qtukey(p = 1 - alfa, nmeans = t, df = df_res)
hsd_val <- q_crit * sqrt(mse_adj / t)

cat("--- PARÁMETROS TUKEY ---\n")
cat(sprintf("Estadístico q crítico: %.4f\n", q_crit))
cat(sprintf("Límite Crítico HSD: %.4f\n\n", hsd_val))

# Función dinámica para calcular pares y diagnosticar
tukey_eval <- function(factor_name) {
  medias <- tapply(df$Horas, df[[factor_name]], mean)
  nombres <- names(medias)
  pares <- combn(nombres, 2)
  
  cat(sprintf("--- COMPARACIONES MÚLTIPLES: %s ---\n", toupper(factor_name)))
  resultados <- data.frame(Par = character(), Diferencia_Abs = numeric(), Diagnostico = character())
  
  for(i in 1:ncol(pares)) {
    p1 <- pares[1, i]
    p2 <- pares[2, i]
    diff_abs <- abs(medias[p1] - medias[p2])
    diagnostico <- ifelse(diff_abs > hsd_val, "SIGNIFICATIVO", "No Significativo")
    
    resultados <- rbind(resultados, data.frame(
      Par = paste(p1, "vs", p2),
      Diferencia_Abs = round(diff_abs, 4),
      Diagnostico = diagnostico
    ))
  }
  print(resultados, row.names = FALSE)
  cat("\n")
}

# Ejecutar para TODOS los factores del diseño
tukey_eval("Algoritmo")
tukey_eval("Linea")
tukey_eval("Turno")

import pandas as pd
import numpy as np
from scipy import stats
import itertools

# 1. CONSTRUCCIÓN DE LA MATRIZ DE DATOS
data = {
    'Linea': np.repeat(['L1', 'L2', 'L3', 'L4'], 4),
    'Turno': np.tile(['T1', 'T2', 'T3', 'T4'], 4),
    'Algoritmo': ['A', 'B', 'C', 'D',
                  'B', 'C', 'D', 'A',
                  'C', 'D', 'A', 'B',
                  'D', 'A', 'B', 'C'],
    'Horas': [24.47, 19.69, 17.04, 22.82,
              21.63, 17.32, 24.75, 25.24,
              16.75, 21.54, np.nan, 20.85,  # L3, T3, Alg A es el dato perdido
              23.25, 23.54, 21.66, 18.87]
}
df = pd.DataFrame(data)

# 2. ESTIMACIÓN ANALÍTICA DEL DATO PERDIDO (Mínimos Cuadrados DCL)
t = 4
idx_na = df['Horas'].isna()
na_linea = df.loc[idx_na, 'Linea'].values[0]
na_turno = df.loc[idx_na, 'Turno'].values[0]
na_alg = df.loc[idx_na, 'Algoritmo'].values[0]

# Sumas parciales sin el dato faltante
R_sum = df.loc[(df['Linea'] == na_linea) & ~idx_na, 'Horas'].sum()
C_sum = df.loc[(df['Turno'] == na_turno) & ~idx_na, 'Horas'].sum()
T_sum = df.loc[(df['Algoritmo'] == na_alg) & ~idx_na, 'Horas'].sum()
G_sum = df.loc[~idx_na, 'Horas'].sum()

# Aplicar la fórmula DCL: Y^ = [t(R+C+T) - 2G] / [(t-1)(t-2)]
y_hat = (t * (R_sum + C_sum + T_sum) - 2 * G_sum) / ((t - 1) * (t - 2))
df.loc[idx_na, 'Horas'] = y_hat

print("--- DATO PERDIDO ESTIMADO ---")
print(f"Línea 3, Turno 3, Algoritmo A = {y_hat:.4f}\n")

# 3. ANÁLISIS DE VARIANZA (ANOVA)
# Factor de corrección y Suma Total de Cuadrados
N = t * t
G_total = df['Horas'].sum()
FC = (G_total ** 2) / N

SS_Total = (df['Horas'] ** 2).sum() - FC
SS_Trt = (df.groupby('Algoritmo')['Horas'].sum() ** 2).sum() / t - FC
SS_Row = (df.groupby('Linea')['Horas'].sum() ** 2).sum() / t - FC
SS_Col = (df.groupby('Turno')['Horas'].sum() ** 2).sum() / t - FC
SS_Error = SS_Total - SS_Trt - SS_Row - SS_Col

# Grados de Libertad (Ajustado por dato perdido: se resta 1 adicional al Total y al Error)
DF_Trt = t - 1
DF_Row = t - 1
DF_Col = t - 1
DF_Total = N - 1 - 1
DF_Error = DF_Total - DF_Trt - DF_Row - DF_Col

# Cuadrados Medios
MS_Trt = SS_Trt / DF_Trt
MS_Row = SS_Row / DF_Row
MS_Col = SS_Col / DF_Col
MS_Error = SS_Error / DF_Error

# Estadísticos F0
F_Trt = MS_Trt / MS_Error
F_Row = MS_Row / MS_Error
F_Col = MS_Col / MS_Error

# Valores P (usando scipy.stats.f.sf para la cola derecha de la distribución F)
p_Trt = stats.f.sf(F_Trt, DF_Trt, DF_Error)
p_Row = stats.f.sf(F_Row, DF_Row, DF_Error)
p_Col = stats.f.sf(F_Col, DF_Col, DF_Error)

# Construir Tabla ANOVA
anova_data = {
    'Fuente': ['Algoritmos', 'Líneas', 'Turnos', 'Error', 'Total'],
    'GL': [DF_Trt, DF_Row, DF_Col, DF_Error, DF_Total],
    'SC': [SS_Trt, SS_Row, SS_Col, SS_Error, SS_Total],
    'CM': [MS_Trt, MS_Row, MS_Col, MS_Error, np.nan],
    'F0': [F_Trt, F_Row, F_Col, np.nan, np.nan],
    'Valor-P': [p_Trt, p_Row, p_Col, np.nan, np.nan]
}
anova_df = pd.DataFrame(anova_data)

print("--- TABLA ANOVA (AJUSTADA POR DATO PERDIDO) ---")
print(anova_df.to_string(index=False, na_rep='', float_format="%.4f"))
print("\n")

# 4. PRUEBAS POST-HOC DE TUKEY EXHAUSTIVAS
alpha = 0.05
# Extraer valor q_crítico de la distribución de rango estudentizado
q_crit = stats.studentized_range.ppf(1 - alpha, k=t, df=DF_Error)
hsd_val = q_crit * np.sqrt(MS_Error / t)

print("--- PARÁMETROS TUKEY ---")
print(f"Estadístico q crítico: {q_crit:.4f}")
print(f"Límite Crítico HSD: {hsd_val:.4f}\n")

def tukey_eval(factor_name):
    print(f"--- COMPARACIONES MÚLTIPLES: {factor_name.upper()} ---")
    medias = df.groupby(factor_name)['Horas'].mean()
    niveles = medias.index.tolist()
    # Generar todos los pares posibles k(k-1)/2
    pares = list(itertools.combinations(niveles, 2))
    
    resultados = []
    for p1, p2 in pares:
        diff_abs = abs(medias[p1] - medias[p2])
        diagnostico = "SIGNIFICATIVO" if diff_abs > hsd_val else "No Significativo"
        resultados.append({
            'Par': f"{p1} vs {p2}",
            'Diferencia_Abs': diff_abs,
            'Diagnostico': diagnostico
        })
    
    res_df = pd.DataFrame(resultados)
    print(res_df.to_string(index=False, float_format="%.4f"))
    print("\n")

# Ejecutar para TODOS los factores del diseño experimental
tukey_eval('Algoritmo')
tukey_eval('Linea')
tukey_eval('Turno')

Recomendaciones sobre algoritmos de mantenimiento predictivo
🤖 Algoritmo (Tratamiento)

Se rechaza H₀ (p ≈ 3.5e-5). El algoritmo C (LSTM) produce significativamente menos horas de inactividad (media ≈ 18.54 h), seguido de D (Transformer) ≈ 21.20 h. Los algoritmos A (Reglas) y B (Random Forest) son estadísticamente equivalentes y los de peor desempeño (≈22.97 h y 22.54 h). La prueba de Tukey confirma que C es superior a todos, y D también es mejor que A y B.

🏭 Línea y Turno (Bloques)

Ambos bloques resultaron significativos (p < 0.05). La Línea 3 muestra la menor inactividad, mientras que el Turno 4 es el más problemático. Controlar estos factores fue crucial para aislar el efecto del algoritmo.

🧮 Consistencia entre softwares y dato faltante

Minitab, Excel, R y Python coinciden en la significancia de los factores, aunque Minitab no resta el grado de libertad por el dato perdido (GLError = 6 vs 5 en los demás). Esta discrepancia metodológica no altera las conclusiones prácticas.

🏆 Recomendación final

Se recomienda implementar el algoritmo C (LSTM) como solución de mantenimiento predictivo, ya que reduce las horas de inactividad en aproximadamente 4.4 horas respecto al promedio de A y B. Dado que cada hora de inactividad equivale a USD 42,000, el ahorro mensual estimado supera los USD 184,000. Si los recursos computacionales son limitados, el algoritmo D (Transformer) es una alternativa válida, aunque menos eficiente que LSTM. Se recomienda además investigar las causas del alto tiempo de inactividad en el Turno 4.
